# F5: Smart Error Hints (Reactive) for Web Scraping

## Findings

### [1] Scrapling's adaptive feature uses a two-phase save/match protocol with 12-factor structural similarity scoring to relocate broken selectors
- quote: "Adaptive scraping works in two phases: 1. Save Phase: Store unique properties of elements 2. Match Phase: Find elements with similar properties later"
- url: https://scrapling.readthedocs.io/en/latest/parsing/adaptive/
- source_type: primary
- published: 2026 (docs current as of v0.4.11)
- confidence: high

### [2] Scrapling's Rust port (v0.2.0, July 2026) implements a 12-factor scoring algorithm using `strsim` crate for element relocation
- quote: "The similarity score is computed across up to 12 equally-weighted factors: Tag name (exact match 0 or 1), Text content (SequenceMatcher ratio), Attributes (dict_diff), class/id/href/src attributes, Ancestor path, Parent tag, Parent attributes, Parent text, Sibling tags. Final score: `(sum / checks) * 100`"
- url: https://docs.rs/scrapling/0.2.0/scrapling/adaptive/index.html
- source_type: primary
- published: 2026-07-09
- confidence: high

### [3] Scrapling stores element fingerprints (tag, text, attributes, siblings, path, parent info) in SQLite and matches candidates using configurable threshold (default 40%)
- quote: "The unique elements we are relying on are: Element tag name, text, attributes (names and values), siblings (tag names only), and path (tag names only). Element's parent tag name, attributes (names and values), and text."
- url: https://scrapling.readthedocs.io/en/latest/parsing/adaptive/
- source_type: primary
- published: 2026
- confidence: high

### [4] Scrapling's auto_save pattern lets selectors automatically persist element fingerprints for future adaptive recovery
- quote: "element = page.css('#p1', auto_save=True) ... element = page.css('#p1', adaptive=True) # Scrapling still finds it!"
- url: https://scrapling.readthedocs.io/en/latest/parsing/adaptive/
- source_type: primary
- published: 2026
- confidence: high

### [5] Scrapling adaptive fails silently when tag types change (article→div, h2→h3) — returns 0 matches at every threshold with no warning signal
- quote: "Adaptive returns 0 matches at every threshold from 10 to 90 when tag type changes, even with class rename being a hint... Add an explicit tag-agnostic mode argument like ignore_tag=True... Surface a warning when adaptive returns 0 from a saved identifier so users know the algorithm gave up rather than the page being empty"
- url: https://github.com/D4Vinci/Scrapling/issues/260
- source_type: community
- published: 2026-05-02
- confidence: high

### [6] Scrapling includes a built-in MCP server for AI-assisted scraping, pre-filtering content before passing to LLM agents like Claude/Cursor
- quote: "MCP Server to be used with AI: Built-in MCP server for AI-assisted Web Scraping and data extraction. The MCP server features powerful, custom capabilities that leverage Scrapling to extract targeted content before passing it to the AI, thereby speeding up operations and reducing costs by minimizing token usage."
- url: https://github.com/D4Vinci/Scrapling
- source_type: primary
- published: 2026-07-08
- confidence: high

### [7] DrissionPage returns a falsy `NoneElement` object instead of throwing when selectors fail, enabling clean `if`-based error handling
- quote: "默认情况下，找不到元素时不会立即抛出异常，而是返回一个NoneElement对象。这个对象用if判断表现为False，调用其功能会抛出ElementNotFoundError异常。这样可以用if判断是否找到元素，也可以用try去捕获异常。"
- url: https://github.com/leonvking0/drisson-page-docs/blob/main/browser_control/get_elements/behavior.md
- source_type: primary (translated from Chinese docs)
- published: 2024-2025
- confidence: high

### [8] DrissionPage provides three configurable failure modes: NoneElement (default), global raise, or custom fallback value via `set_NoneElement_value()`
- quote: "Settings.set_raise_when_ele_not_found(True) ... tab.set.NoneElement_value('没找到')"
- url: https://github.com/leonvking0/drisson-page-docs/blob/main/browser_control/get_elements/behavior.md
- source_type: primary (translated from Chinese docs)
- published: 2024-2025
- confidence: high

### [9] DrissionPage has built-in fuzzy matching (`:` for contains, `=` for exact) and 10-second auto-wait on all element searches
- quote: "The colon: indicates a fuzzy match, and the equal sign = indicates an exact match... The element search timeout is 10 seconds by default."
- url: https://github.com/g1879/DrissionPage
- source_type: primary
- published: 2024
- confidence: high

### [10] DrissionPage has automatic retry on connection errors (default 2 retries) with configurable retry count and interval per request
- quote: "If there is an error in the connection, the program will automatically retry twice. The number of retries and the waiting interval can be specified. page.get(url, retry, interval, **kwargs)"
- url: https://pypi.org/project/DrissionPage/1.7.3/
- source_type: primary
- published: 2025
- confidence: high

### [11] An LLM-based self-healing system (2026) uses a local model to propose CSS/XPath replacements, validates candidates against live HTML and type schemas, and writes fixes to DB without redeploy
- quote: "The Self-Healer publishes a repair job to a Redis queue. A Python sidecar picks up that job, fetches the current HTML, trims it to fit a token budget, and sends a targeted prompt to a local LLM. The LLM proposes new CSS/XPath selector candidates with confidence scores. Each candidate is then tested against the live HTML using BeautifulSoup and lxml."
- url: https://dev.to/viniciuspuerto/when-the-scraper-breaks-itself-building-a-self-healing-css-selector-repair-system-312d
- source_type: secondary
- published: 2026-04-01
- confidence: medium

### [12] Scrapling's `find_similar()` method locates elements with same tree depth, tag name, and parent context — useful for discovering sibling elements from one known element
- quote: "Find elements that are in the same tree depth in the page with the same tag name and same parent tag etc... then return the ones that match the current element attributes with a percentage higher than the input threshold."
- url: https://scrapling.readthedocs.io/en/latest/api-reference/selector.html
- source_type: primary
- published: 2026
- confidence: high

## Dead ends
- DrissionPage does NOT have an equivalent to Scrapling's adaptive/fuzzy relocation — it has no similarity-based element recovery when selectors break
- No official Scrapling docs describing the exact weights or priority of the 12 factors in the Rust port's scoring algorithm (the Python implementation also lacks documentation of internal weights)
- No academic papers found specifically comparing adaptive selector recovery approaches in 2024-2026

## Suggested follow-ups
- How does the `scrapling-rs` Rust port's `calculate_similarity_score` handle the weighting across its 12 factors — is the "12 equally-weighted" claim accurate, or does the source code use different weights?
- What is the actual SQLite schema used by Scrapling's adaptive storage, and how can a Rust scraper replicate the save/match lifecycle?
- Could Scrapling's 12-factor structural similarity approach be combined with DrissionPage's NoneElement pattern for a layered error recovery system in webfang?
