# F3: Vision Grounding (Set-of-Mark) for Web Scraping

## Findings

### [1] Set-of-Mark (SoM) prompting enables GPT-4V to outperform fine-tuned grounding models in zero-shot settings
- quote: "GPT-4V with SoM in zero-shot setting outperforms the state-of-the-art fully-finetuned referring expression comprehension and segmentation model on RefCOCOg."
- url: https://arxiv.org/abs/2310.11441
- source_type: primary
- published: 2023-10-17
- confidence: high

### [2] Anthropic Computer Use tool adds 466-499 tokens to system prompt plus 735 tokens per tool definition, creating fixed overhead per screenshot round-trip
- quote: "The computer use beta adds 466-499 tokens to the system prompt... Claude 4.x models: 735 tokens" (per tool definition)
- url: https://platform.claude.com/docs/en/agents-and-tools/tool-use/computer-use-tool
- source_type: primary
- published: 2025 (updated 2025-11-24 beta header)
- confidence: high

### [3] Claude Opus 4.7 screenshots cost ~$0.007 per web-resolution image via area-based tokenization (w×h/750), 2.7x more expensive than GPT-5.5 for the same 1024×1024 image
- quote: "Claude Opus 4.7: Image tokens approximate (width × height) / 750. The long edge is capped at 2,576 pixels... Web image (1024×1024): Claude = $0.00699, GPT-5.5 = $0.00512, Gemini = $0.00206"
- url: https://blog.roboflow.com/image-token-cost-vlm
- source_type: secondary
- published: 2026-05-04
- confidence: high

### [4] Browserbase's Stagehand defaults to accessibility tree extraction, not screenshots — visual extract (screenshot: true) is opt-in and only supported with AI SDK clients
- quote: "Set `screenshot: true` when the extraction needs visual information from the current viewport in addition to the page accessibility tree... `screenshot: true` captures the current viewport, not the full page. It is only supported with AI SDK clients."
- url: https://docs.stagehand.dev/v3/basics/extract.md
- source_type: primary
- published: unknown
- confidence: high

### [5] Browserbase/Stagehand integrates Set-of-Mark prompting for visual grounding — interactive elements are highlighted as numbered marks on the browser viewport
- quote: "interactive elements are highlighted as a Set of Marks to improve action grounding by VLMs (as shown in Figure 3), and before each action, relevant elements are highlighted with explanatory text boxes containing natural language descriptions."
- url: https://aclanthology.org/2025.naacl-demo.36.pdf
- source_type: primary
- published: 2025-04-30 (NAACL 2025)
- confidence: high

### [6] Stagehand's observe() returns structured actions with selectors (not coordinates), making it DOM-grounded rather than pixel-grounded — a key difference from Anthropic Computer Use
- quote: "[{ description: 'Learn more button', method: 'click', arguments: [], selector: 'xpath=/html[1]/body[1]/shadow-demo[1]//div[1]/button[1]' }]"
- url: https://docs.stagehand.dev/v3/basics/observe.md
- source_type: primary
- published: unknown
- confidence: high

### [7] Anthropic Computer Use explicitly warns that latency is "too slow" for human-AI interactions and recommends background/information-gathering use cases
- quote: "Latency: The current computer use latency for human-AI interactions might be too slow compared to regular human-directed computer actions. Focus on use cases where speed isn't critical (for example, background information gathering, automated software testing) in trusted environments."
- url: https://platform.claude.com/docs/en/agents-and-tools/tool-use/computer-use-tool
- source_type: primary
- published: 2025
- confidence: high

### [8] Browserbase costs 4.4x more and runs 6.7x slower than Browser Use for an equivalent multi-page scraping task (Hacker News 20-article extraction: $1.46 vs $0.33, 401s vs 60s)
- quote: "Browserbase cost 4.4x more and took 6.7x longer."
- url: https://browser-use.com/posts/web-scraping-guide-2026
- source_type: secondary
- published: 2026-03-26
- confidence: medium (competitor comparison, but task was identical)

### [9] Anthropic Computer Use 20251124 beta adds zoom capability for small UI elements — specifically designed for screenshots where text/UI is too small at default resolution
- quote: "zoom: View a specific screen region at full resolution. Requires `enable_zoom: true` in tool definition. Takes a `region` parameter with coordinates [x1, y1, x2, y2]."
- url: https://platform.claude.com/docs/en/agents-and-tools/tool-use/computer-use-tool
- source_type: primary
- published: 2025-11-24
- confidence: high

### [10] LiteWebAgent demonstrates a decoupled architecture where SoM is used for action grounding (translating natural language actions into Playwright selectors), while action generation uses the trajectory history without the full observation
- quote: "We employ a tailored prompting method that leverages the current webpage observation ot as context to transform the natural language action at into executable actions, as a wrapper of Playwright code... This decoupled approach... significantly reducing the number of prompt tokens required for action generation."
- url: https://aclanthology.org/2025.naacl-demo.36.pdf
- source_type: primary
- published: 2025-04-30
- confidence: high

### [11] Browserbase pricing (June 2026): Developer plan at $20/mo includes browser hours, with overages at ~$0.12/hour; the Extract API meter is separate from browser hours at ~$4/1,000 calls without proxies
- quote: "Developer overages include browser hours around $0.12 per hour... Extract API overages around $4 per 1,000 calls without proxies"
- url: https://scrapegraphai.com/blog/browserbase-pricing
- source_type: secondary
- published: 2026-06
- confidence: medium (third-party analysis, pricing may change)

### [12] Anthropic Computer Use with Claude Sonnet 4.6/Opus 4.6 is recommended at medium thinking effort for UI tasks — low effort uses fewer output tokens than disabling thinking entirely due to fewer retries
- quote: "Claude Sonnet 4.6 and Claude Opus 4.6: use `medium` as the default (best accuracy-to-cost ratio). Avoid `max`, which adds token cost without improving accuracy on UI tasks. On these models, `low` uses fewer output tokens than disabling thinking entirely (fewer mistakes mean fewer retries)."
- url: https://platform.claude.com/docs/en/agents-and-tools/tool-use/computer-use-tool
- source_type: primary
- published: 2025
- confidence: high

## Dead ends
- Browserbase official docs did not expose detailed SoM/DOM marker implementation specifics (how marks are overlaid, which segmentation model is used); the NACL 2025 LiteWebAgent paper is the best public source on this.
- Anthropic's vision pricing page returned 503; cost figures were sourced from the Roboflow comparison blog (May 2026) cross-referenced with Anthropic's documented tokenization formula.
- Search for "Browserbase DOM marker" implementation details hit rate limits; Stagehand docs focus on API surface, not internals.
