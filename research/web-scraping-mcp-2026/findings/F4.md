# F4: Semantic DOM Pruning

## Findings

### [1] Jina Reader uses Mozilla Readability as its initial content extraction step
- quote: "Mozilla's Readability package is leveraged to extract the main content, removing elements like headers, footers, navigation bars, and sidebars."
- url: https://ritvik19.medium.com/papers-explained-221-reader-lm-7382b9eb6ed9
- source_type: secondary
- published: Sep 30, 2024
- confidence: high

### [2] Jina Reader API returns markdown by default with extensive header-based control
- quote: "The Reader API uses a proxy to fetch any URL, rendering its content in a browser to extract high-quality main content."
- url: https://jina.ai/reader
- source_type: primary
- published: unknown
- confidence: high

### [3] Reader-LM v2 is a 1.5B parameter SLM for HTML-to-Markdown conversion with 256K token context
- quote: "ReaderLM-v2 is our latest small language model (SLM) for converting raw HTML into clean markdown or JSON. It supports documents up to 512K tokens across 29 languages and offers 20% higher accuracy compared to its predecessor."
- url: https://jina.ai/reader
- source_type: primary
- published: unknown
- confidence: high

### [4] Prune4Web achieves 25-50x DOM reduction via LLM-generated Python scoring programs
- quote: "We introduce Prune4Web, a novel paradigm that transforms DOM processing from LLM-based filtering to programmatic pruning. Our key innovation is DOM Tree Pruning Programming, where an LLM generates executable Python scoring programs to dynamically filter DOM elements."
- url: https://arxiv.org/html/2511.21398v1
- source_type: primary
- published: Nov 26, 2025
- confidence: high

### [5] Prune4Web increases grounding accuracy from 46.8% to 88.28% through programmatic DOM pruning
- quote: "On our low-level task grounding task, our approach dramatically increases grounding accuracy from 46.8% to 88.28%, highlighting its effectiveness."
- url: https://arxiv.org/html/2511.21398v1
- source_type: primary
- published: Nov 26, 2025
- confidence: high

### [6] Trafilatura outperforms Mozilla Readability in extraction benchmarks
- quote: "Trafilatura (standard) | F-Score: 0.909 | Precision: 0.914 | Recall: 0.904 ... Readability-lxml | F-Score: 0.801 | Precision: 0.891 | Recall: 0.729"
- url: https://dev.to/stevengonsalvez/browser-tools-for-ai-agents-part-4-skip-the-browser-save-80-on-tokens-304c
- source_type: secondary
- published: Apr 26, 2026
- confidence: high

### [7] SentienceAPI prunes ~95% of DOM nodes using TreeWalker in Rust/WASM
- quote: "It uses a TreeWalker to traverse the DOM and ruthlessly prune ~95% of the nodes. It drops wrapper divs, invisible elements, scripts, and layout noise before it leaves the browser."
- url: https://www.reddit.com/r/LocalLLaMA/comments/1qcxllu/i_built_a_dompruning_engine_to_run_reliable
- source_type: community
- published: Jan 15, 2026
- confidence: medium

### [8] Jina Reader API supports CSS selector-based extraction and exclusion
- quote: "x-target-selector — a CSS selector. Reader returns content within the matched element instead of the full page. Useful when automatic content extraction misses what you want."
- url: https://github.com/jina-ai/reader
- source_type: primary
- published: unknown
- confidence: high

### [9] Reader-LM training uses Jina Reader API output as ground truth
- quote: "A data pipeline is built to ensure only high-quality markdown entries were included in the training set, utilizing Jina Reader API."
- url: https://ritvik19.medium.com/papers-explained-221-reader-lm-7382b9eb6ed9
- source_type: secondary
- published: Sep 30, 2024
- confidence: high

### [10] Cloudflare markdown.new converts URLs to markdown at CDN layer with 82% token reduction
- quote: "Raw HTML: 9,541 tokens. Cleaned markdown: 1,678 tokens. That's an 82% reduction. On a single page."
- url: https://dev.to/stevengonsalvez/browser-tools-for-ai-agents-part-4-skip-the-browser-save-80-on-tokens-304c
- source_type: secondary
- published: Apr 26, 2026
- confidence: high
